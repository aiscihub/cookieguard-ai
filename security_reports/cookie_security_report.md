## Key Findings

### 1 — Authentication Cookie Security Is Weak

Of all authentication cookies found:

- Only **73.5%** carried the `Secure` flag, meaning **26.5%** can be transmitted over unencrypted HTTP connections.
- Only **64.4%** carried the `HttpOnly` flag, meaning **35.6%** are readable by JavaScript — exposing them to XSS attacks.

A cookie missing both flags is trivially stealable via a single injected script or a passive network observer on public Wi-Fi.

## Methodology

### Data Collection
Each site was visited once using a headless Chromium browser (Playwright). The browser waited 2 seconds after page load to allow JavaScript-set cookies to be established. All cookies present in the browser context after the visit were recorded.

### Classification
Cookies were classified into four types using CookieGuard AI's rule-based classifier:
- **Authentication** — session tokens, login identifiers, CSRF tokens
- **Tracking** — analytics, advertising, behavioral profiling
- **Preference** — language, theme, locale settings
- **Other** — unclassified

### Risk Scoring
Authentication cookies were scored on a 0–100 scale based on:
- Missing `Secure` flag: +30 points
- Missing `HttpOnly` flag: +25 points
- Missing/weak `SameSite`: +15 points
- Wildcard domain scope: +10 points
- Long expiration (>1 year): +15 points

Scores map to: Critical (≥40), High (≥25), Medium (≥10), Low (>0).

### Privacy
No user accounts were logged into. No personal data was collected. Only cookie metadata (name, domain, security attributes) was recorded — no cookie values were stored.

## Implications

The data reveals a significant gap between the security posture users expect and what websites actually implement:

1. **Users on public Wi-Fi are at real risk.** With 26.5% of authentication cookies lacking the `Secure` flag, a passive network attacker can harvest session tokens from unencrypted requests.

2. **XSS attacks have a large attack surface.** With 35.6% of session cookies readable by JavaScript, a single stored XSS vulnerability on a page is sufficient to extract every session token.

3. **Tracking is the norm, not the exception.** With 98.7% of sites running trackers, users have virtually no way to browse major websites without being observed by ad-tech companies — unless they use a tool like CookieGuard AI.

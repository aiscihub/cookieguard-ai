Methodology
🌐
Data Collection
Each site visited once via headless Chromium (Playwright). Browser waits 2s post-load for JS-set cookies. No cookie values stored.
🤖
AI Classification
CookieGuard classifier categorizes each cookie: Authentication, Tracking, Preference, or Other — using name patterns, domain matching, and value entropy.
📊
Risk Scoring
Auth cookies scored 0–100: missing Secure (+30), missing HttpOnly (+25), weak SameSite (+15), wildcard domain (+10), long expiry (+15).
Implications
1
Users on public Wi-Fi are at real risk. With 26.5% of authentication cookies lacking the Secure flag, a passive observer on any unencrypted network can harvest live session tokens.
2
XSS attacks have a massive attack surface. With 35.6% of session cookies readable by JavaScript, a single stored XSS vulnerability is sufficient to extract every session token on a page.
3
Tracking is the default, not the exception. With 97.7% of sites running trackers and an average of 9.4 companies per site, users cannot browse without being profiled — unless they use CookieGuard AI.
